# FlowMate

AI 陪伴式心流助手 - 让专注变得更简单

## 一、产品定义与创意本源

### （一）产品定位
面对长时间面对电脑的数字工作者，解决被番茄钟计时结束、消息等打断的问题，我们的FlowMate可以帮助你们进入心流及维持心流状态。不同于大部分番茄钟，我们用更高端的方法识别你当时的专注状态，从而灵活调整专注时长。同时，还可以在专注过程中帮你记录突然出现的灵感或较小的待办事项，以更好的维护你的专注状态。

### （二）核心痛点及用户画像

| 画像 | 典型标签 | 行为特征 | 核心痛点 | 核心需求 |
| :--- | :--- | :--- | :--- | :--- |
| **重度脑力劳动者** | 熬夜修Bug的程序员、产品经理 | 长时间面对电脑，需要极高的逻辑专注力；经常多线任务并行。 | 1. **任务瘫痪**：面对庞大复杂的代码或文档，不知从何下手，迟迟无法启动<br>2. **灵感焦虑**：深度工作时总会冒出“某个Bug的解法”等杂念，不记怕忘，记又打断思路。 | 效率、专注、无缝衔接 |

## 二、核心交互体验

### （一）伴（拟真在场）
通过状态同步（同步翻书、敲键盘等）和眼神交互，营造真实的“社会性在场感”，提供无压力的视觉陪伴，在潜意识中本能地带动用户进入专注状态。

### （二）进（智能助推）
在用户因任务大而陷入启动困难时主动介入，提供微型任务拆解；在灵感迸发时，利用“闪念记忆”以最低交互成本实现手眼不离屏幕的无感记录。

### （三）守（心流保护）
颠覆传统时间管理，独创“动态降维”与“心流豁免”机制。既能根据眨眼频率和屏幕使用状态等实时状态缩短专注时长以保护长期动力；又能在识别到高专注状态时主动拦截闹钟与干扰，守护用户心流。

### （四）赏（价值显化）
将无形的努力转化为可视化的“心流脉冲图”与随时可以查看的“心流日记”，缓解延迟反馈带来的焦虑感。

## 三、技术架构

### （一）视听层
1. **端云混合的屏幕检测**：采用独创的“双重过滤机制”。在本地客户端，利用OpenCV进行64×64缩略图级别的“视觉防抖”差值计算，拦截静默无效画面。仅当发生实质性变更且突破“时间冷却”阈值时，才调用云端 Qwen-VL-Max模型进行屏幕内容的语义级深度识别与判断来确定工作与摸鱼的边界。
2. **眨眼频率监测**：通过本地OpenCV Haar cascades算法实时提取眼睑闭合频率从而推断专注度，构建结构化的疲劳指数，全程在本地闭环以保障隐私。
3. **抗噪听觉流**：集成SenseVoice模型，负责精准捕获并转录复杂环境或伴随键盘敲击声中的指令。

### （二）认知层
1. **分层模型的多智能体中枢**：由Qwen-Max作为高复杂决策大脑，同时引入轻量级模型承担低成本、高频次的基础认知任务。系统接收语音文本、屏幕语义结果、交互状态等多模态输入后，先由轻量模型完成意图初筛与简单分类；当检测到任务歧义高、上下文复杂、需要多轮推理或情感化生成时，再自动升级调用Qwen-Max进行深度判断与生成，从而在响应速度、调用成本与推理质量之间实现动态平衡。
2. **微型任务拆解引擎**：针对“任务瘫痪”场景，LLM被严格Prompt约束，将庞大模糊的任务直接转化为包含3个极简物理步骤的JSON数组。该结构化Payload被送入前端，直接渲染为可交互的Checklist。

### （三）节奏层
* **多维数据仲裁与心流豁免**：结合前端的全局键盘敲击率监听与摄像头的眨眼频率数据。当倒计时归零时，若系统判断用户正处于高频输入或极高专注状态，将自动触发“心流豁免”逻辑：系统将在底层静默拦截闹钟触发，并将UI无缝切换为正计时模式，保持注意力的连续性。

### （四）表达层
1. **音动一体化协议**：云端调用CosyVoice-Chat/Instruct模型生成具备情绪色彩（如严肃监督、温和鼓励）的音频流。该协议在输出音频的同时，会同步下发情绪触发指令。当语音开始播放时，虚拟人进入说话状态；当一句话结束但下一句话尚未到来时，角色会保持说话准备姿态，而不是立刻切回静止状态，从而减少动作跳变，提升表达的连贯性和自然感。
2. **流式语音输出机制**：采用“边生成、边合成、边播放”的方式，每当LLM生成一句相对完整的话，系统就立即触发TTS合成，并通过WebSocket实时发送到前端，从而明显缩短用户从“提问”到“听见回答”的等待时间。
3. **句级切分与顺序播放**：针对较长回复，系统会自动按句子进行切分，并将每一句分别合成为独立音频块。前端按顺序连续播放这些音频块，使虚拟人能够像真人一样“边想边说”，避免一次性等待整段音频生成完成所造成的卡顿。

### （五）安全层
* **桌面隐私重于一切**：核心视觉预处理与静默画面过滤均在本地客户端闭环完成。AI仅在实质性变更时脱敏上云，实现本地视觉闭环。

## 四、竞品分析与产品总结

| 竞品类别 | 代表产品 | 感知维度 | 干预方式 | FlowMate-Echo核心壁垒 |
| :--- | :--- | :--- | :--- | :--- |
| **传统时间管理/番茄钟** | Forest、番茄To-Do、潮汐等 | 无感知（完全依赖用户手动设定固定时长） | 机械式打断 | **颠覆机械计时**：独创“心流豁免” ，在识别到高专注状态时主动拦截闹钟；疲劳时则启用“动态降维”缩短目标。 |
| **虚拟自习室/桌宠** | StudyStream、放松时光等 | 极弱感知（单向的视频流展示或简单的点击互动，无法获知用户真实状态） | 静态氛围陪伴 | **从被动到主动**：具备端云混合架构的“眼”，能通过状态同步、眼神交互提供真正具有“社会性在场感”的主动陪伴。 |
| **屏幕监控/防沉迷** | Offscreen、小鸡专注等带有锁定应用功能的产品 | 浅层数据（基于系统进程名或网页URL进行简单分类） | 粗暴拦截 | **讲道理的柔性干预**：引入大模型进行智能屏幕识别，通过对比屏幕内容与当前任务的语义匹配度进行精准仲裁，拒绝死板的一刀切。 |
| **AI效率助手** | Notion AI、ChatGPT等 | 被动输入（依赖用户主动打字、截图或点击按键唤醒） | 回答式响应 | **无感记录与闪念记忆**：通过“微型任务拆解”主动介入用户的“任务瘫痪”；利用“闪念胶囊”实现口述记录，手眼不离屏幕。 |

> **产品总结**
> FlowMate-Echo是一个能读懂你实时状态的桌面视讯伙伴，它会在你卡壳时主动拆解任务帮你破局，在疲惫时柔性调整专注时长为你减压，并在你极度专注时自动拦截闹钟守护你的心流。总之，它是一个懂你节奏、知进退的AI心流伙伴。

如果你需要更完整的技术说明，请查看 `allaboutit.md`。

## 快速开始

### 1. 安装依赖

```bash
npm install
pip install -r requirements.txt
```

### 2. 配置环境变量

复制 `.env.example` 为 `.env` 并填写 API Key：

```env
MODELSCOPE_TOKEN=
DASHSCOPE_API_KEY=your_key
```

> 注意：请勿提交真实 `.env`。

### 3. 启动服务

```bash
# 启动后端
npm run start:backend

# 启动前端（Electron + Vite）
npm run dev
```

## 常见问题

**Q: 数字人视频不显示？**
A: 确保 `src/renderer/assets/video_loops/` 有对应视频文件。

**Q: 后端连接失败？**
A: 确认后端在 8000 端口运行，且 CSP 允许访问本地接口。

**Q: ASR 没有结果？**
A: SenseVoice 录音识别需要 OSS 配置，否则会返回空文本。
